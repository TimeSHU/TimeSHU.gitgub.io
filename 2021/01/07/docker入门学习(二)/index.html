CMD 和 ENTRYPOINT区别

CMD

构建后，当我们执行 docker run 32bb736dabe2 ls -la 时命令正常运行

当我们执行docker run 32bb736dabe2 -ll报错

结论：CMD下会使用-ll替换掉ls -la命令，而不是追加成ls -l-ll命令

ENTRYPOINT

正常运行

正常运行，使用ENTRYPOINT构建的镜像，会将docker run cc21b9ec59dd -ll 追加到命令中，最后执行的命令就是ls -a -ll
